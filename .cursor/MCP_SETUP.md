# Настройка MCP сервера для документации проекта Life

## Описание

MCP сервер `mcp_index.py` предоставляет инструменты для работы с документацией проекта через протокол Model Context Protocol.

## Доступные инструменты

- `search_docs` - Поиск по ключевым словам в документации (`docs/`)
- `search_todo` - Поиск по ключевым словам в TODO документах (`todo/`)
- `get_doc_content` - Получение полного содержимого документа из `docs/`
- `get_todo_content` - Получение полного содержимого документа из `todo/`
- `list_docs` - Список всех документов в `docs/`
- `list_todo` - Список всех документов в `todo/`

## Установка зависимостей

Убедитесь, что установлены все зависимости:

```bash
pip install -r requirements.txt
```

## Настройка в Cursor

Конфигурация MCP сервера находится в `.cursor/mcp.json`.

Если эта конфигурация не подхватывается автоматически, настройте MCP сервер в настройках Cursor:

1. Откройте настройки Cursor (Ctrl+, или Cmd+,)
2. Перейдите в раздел MCP или Model Context Protocol
3. Добавьте новый сервер со следующими параметрами:
   - **Name**: `life-docs`
   - **Command**: `python`
   - **Args**: `["mcp_index.py"]`
   - **Working Directory**: корень проекта (где находится `mcp_index.py`)

## Альтернативная настройка через файл конфигурации Cursor

В корне проекта создан файл `mcp_config_example.json` с примером конфигурации.

Если используется файл конфигурации Cursor (обычно в `~/.cursor/mcp_settings.json` или `%APPDATA%/Cursor/User/globalStorage/mcp.json` на Windows), скопируйте туда содержимое из `mcp_config_example.json` и обновите пути:

**Важно**: В конфигурации замените `D:/Space/life` на актуальный путь к вашему проекту.

### Настройка через UI Cursor

1. Откройте Cursor Settings (Ctrl+, или Cmd+,)
2. Найдите раздел "MCP" или "Model Context Protocol"
3. Добавьте новый сервер:
   - **Name**: `life-docs`
   - **Command**: `python` (или полный путь к Python: `D:\Tools\Py_3.10\python.exe`)
   - **Args**: `["mcp_index.py"]` или `["D:/Space/life/mcp_index.py"]`
   - **Working Directory**: `D:/Space/life` (корень проекта)

## Тестирование

Для проверки работы сервера:

```bash
python mcp_index.py
```

Сервер должен запуститься и ожидать соединения через stdio.

## Примечания

- Сервер использует FastMCP для упрощения реализации
- Документация читается из директории `docs/`
- TODO документы читаются из директории `todo/`
- Поиск выполняется по текстовому совпадению (без учета регистра)
