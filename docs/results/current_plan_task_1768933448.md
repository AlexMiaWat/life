# План выполнения задачи: Обновление мониторинга для показа physical vs subjective time

## Задача
Обновить мониторинг (`src/monitor/console.py`) чтобы показывать physical vs subjective time

## Анализ текущего состояния
- Текущий мониторинг показывает: `возраст: {age:.1f}/{subjective_time:.1f} сек.`
- В проекте есть концепции:
  - `physical_age` (алиас для `age`) - физическое время
  - `subjective_age` (алиас для `subjective_time`) - субъективное время
- Субъективное время рассчитывается на основе состояния системы (энергия, стабильность, интенсивность событий)

## План изменений

### 1. Дизайн нового отображения
- Заменить текущий формат `возраст: {age}/{subjective_time}` на более понятный
- Показывать: `физ: {age:.1f}с | субъект: {subjective_time:.1f}с (x{ratio:.2f})`
- Где `ratio = subjective_time / age` если `age > 0`, иначе `1.0`

### 2. Реализация изменений
- Изменить строку 52 в `src/monitor/console.py`
- Добавить расчет соотношения между временами
- Обновить текстовые метки для лучшей понятности

### 3. Тестирование
- Проверить отображение при разных значениях age и subjective_time
- Убедиться, что логика ratio корректна (не делить на 0)
- Проверить цветовое оформление

## Ожидаемый результат
Мониторинг будет четко показывать:
- Физическое время (секунды)
- Субъективное время (секунды)
- Соотношение между ними (множитель)

Это позволит наблюдателю лучше понимать, как система воспринимает время относительно физического течения.

## Риски
- Потенциальное деление на ноль при age = 0 (нужно обработать)
- Изменение формата может сломать существующие скрипты парсинга логов

## Критерии успеха
- ✅ Мониторинг показывает физическое и субъективное время раздельно
- ✅ Отображается соотношение (ratio) между временами
- ✅ Код остается читаемым и поддерживаемым
- ✅ Логика корректно обрабатывает edge cases (age = 0)
